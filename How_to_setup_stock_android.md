# Latebris 초기 세팅 가이드

이 문서는 **순정 안드로이드 폰에서 Latebris를 처음 테스트할 때 필요한 초기 설정**을 초보자 기준으로 쉽게 설명한 가이드입니다.

핵심만 먼저 말하면 아래와 같습니다.

- **비루팅 순정폰**에서는 앱 설치, 권한 허용, 프리셋 관리, 시뮬레이션 시작/중지, GPS mock 흐름 확인이 가능합니다.
- **Wi-Fi / Cell 값까지 포함한 전체 기능 테스트**는 일반적으로 **루팅 + Magisk + LSPosed**가 필요합니다.
- 즉, 처음에는 **"내 폰이 지금 어느 모드인지"**부터 구분해서 따라가는 것이 가장 중요합니다.

---

## 1. 내 폰이 어떤 상태인지 먼저 구분하기

Latebris는 폰 상태에 따라 가능한 기능이 다릅니다.

### A. 순정폰 + 비루팅

이 경우 보통 아래를 테스트할 수 있습니다.

- 앱 설치 및 실행
- 프리셋 생성 / 수정 / 삭제
- 권한 요청 및 서비스 제어 흐름 확인
- GPS mock 기반 시뮬레이션 흐름 확인

하지만 아래는 보통 되지 않거나 제한적입니다.

- 외부 앱에서 Wi-Fi 결과를 프리셋 값으로 바꾸는 것
- 외부 앱에서 Cell Tower 결과를 프리셋 값으로 바꾸는 것

### B. 순정폰이지만 내가 직접 루팅한 상태

이 경우 루팅만으로는 아직 전체 기능이 완성되지 않습니다.

- 루팅 후에도 **LSPosed 같은 후킹 런타임이 없으면** Wi-Fi / Cell 치환은 동작하지 않습니다.
- 즉, **루팅만 한 상태 = 비루팅보다 준비가 더 된 상태**이지만, 아직 전체 기능 테스트 상태는 아닙니다.

### C. 루팅 + Magisk + LSPosed까지 완료

이 경우 Latebris의 전체 테스트 시나리오에 가장 가깝습니다.

- GPS mock
- Wi-Fi 후킹
- Cell 후킹
- LSPosed scope에 추가한 테스트 대상 앱에서 프리셋 값 확인

---

## 2. 공통 준비물

모든 모드에서 공통으로 필요한 기본 준비입니다.

- Android 12 이상 기기
- Latebris APK
- 테스트할 프리셋
- 위치 권한 허용
- Android 13 이상이면 알림 권한 허용

APK 기본 경로 예시:

- `app/build/outputs/apk/debug/app-debug.apk`

---

## 3. 비루팅 순정폰 초기 세팅

이 섹션은 **"루팅은 안 했지만 일단 앱을 실행해서 기본 기능과 GPS 흐름을 테스트하고 싶다"**는 경우에 맞는 설명입니다.

### 3-1. APK 설치

1. 테스트용 APK를 휴대폰에 복사합니다.
2. APK를 눌러 설치합니다.
3. 설치 차단이 뜨면 브라우저 또는 파일 앱의 **알 수 없는 앱 설치 허용**을 켭니다.

### 3-2. 앱 첫 실행

1. Latebris를 실행합니다.
2. 기본 프리셋이 보이는지 확인합니다.
3. 권한 요청이 보이면 먼저 허용합니다.

### 3-3. 필수 권한 허용

최소한 아래 권한은 허용해야 합니다.

- 위치 권한
- 알림 권한(Android 13 이상)

권한이 없으면 보통 아래 현상이 보입니다.

- Start 버튼이 비활성화됨
- Permission readiness가 준비되지 않은 상태로 표시됨
- Request permissions 버튼이 표시됨

### 3-4. 개발자 옵션 열기

GPS mock 테스트를 하려면 개발자 옵션을 먼저 확인하는 것이 안전합니다.

일반적인 순서는 아래와 같습니다.

1. **설정 → 휴대전화 정보**로 이동
2. **빌드 번호**를 여러 번 누름
3. 개발자 옵션이 활성화되면 다시 설정으로 돌아감
4. **개발자 옵션** 메뉴가 생겼는지 확인

> 제조사에 따라 메뉴 이름이나 위치는 조금 다를 수 있습니다.

### 3-5. 모의 위치 앱 선택 확인

순정폰에서 GPS mock 테스트를 할 때는 아래 설정을 먼저 확인하는 것이 좋습니다.

1. **개발자 옵션**으로 이동
2. **모의 위치 앱 선택(Select mock location app)** 항목 찾기
3. Latebris를 선택할 수 있으면 선택

중요한 점:

- 이 설정은 기기나 OS 버전에 따라 보이는 방식이 조금 다를 수 있습니다.
- 일부 기기에서는 정책이나 OEM 설정에 따라 mock location 동작이 더 엄격할 수 있습니다.

### 3-6. 첫 테스트

1. 앱 실행
2. 프리셋 하나 선택
3. **Start** 누르기
4. 상태가 `Running`으로 바뀌는지 확인
5. 활성 좌표가 표시되는지 확인
6. **Stop** 눌러 `Stopped`로 돌아오는지 확인

### 3-7. 이 단계에서 기대해야 하는 것

비루팅 순정폰에서는 아래를 기대하면 됩니다.

- 앱이 정상적으로 열린다
- 프리셋 편집이 된다
- Start / Stop 흐름이 정상이다
- GPS mock 기반 흐름이 기본적으로 동작한다

반대로 아래는 비정상이라기보다 **제한 사항**일 수 있습니다.

- 외부 앱에서 Wi-Fi 값이 바뀌지 않음
- 외부 앱에서 Cell 값이 바뀌지 않음

## 4. 순정폰이지만 루팅한 경우 초기 세팅

이 섹션은 **기기는 원래 순정 안드로이드였지만, 사용자가 직접 루팅한 경우**를 위한 설명입니다.

### 4-1. 먼저 알아둘 점

루팅은 단순 앱 설정이 아니라 기기 상태를 바꾸는 작업입니다.

보통 아래 리스크가 있습니다.

- 부트로더 언락 과정에서 데이터 초기화 가능
- 보안 정책 변경
- OTA 업데이트 불안정 가능
- 금융/보안 앱 동작 제한 가능

즉, 테스트 전용 기기에서만 진행하는 것이 안전합니다.

### 4-2. 루팅만 했을 때 가능한 것

루팅만 되어 있어도 아래가 자동으로 되는 것은 아닙니다.

- Wi-Fi 후킹 자동 활성화
- Cell 후킹 자동 활성화
- 외부 앱 API 결과 자동 치환

루팅만 한 상태에서는 아래처럼 이해하면 됩니다.

- 비루팅보다 고급 환경으로 갈 준비는 됨
- 하지만 **LSPosed가 없으면 Latebris의 Wi-Fi / Cell 후킹은 아직 완성되지 않음**

### 4-3. 루팅한 폰에서 기본 확인 순서

1. Latebris 설치
2. 위치 / 알림 권한 허용
3. 프리셋 선택
4. Start / Stop 테스트
5. GPS 흐름 확인

여기까지는 비루팅 순정폰과 거의 비슷합니다.

### 4-4. 전체 기능으로 가려면 추가로 필요한 것

Wi-Fi / Cell까지 테스트하려면 보통 아래가 필요합니다.

- Magisk
- LSPosed
- LSPosed에서 Latebris 모듈 활성화
- LSPosed에서 테스트 대상 앱 scope 추가

즉, **루팅 = 중간 단계**, **루팅 + LSPosed = 전체 기능 단계**로 보면 이해가 쉽습니다.

## 5. 루팅 + Magisk + LSPosed 전체 기능 세팅

이 섹션은 Wi-Fi / Cell 치환까지 포함해 Latebris를 제대로 쓰고 싶을 때 따라가는 순서입니다.

### 5-1. 준비 상태 확인

아래가 모두 준비되어 있어야 합니다.

- 루팅 완료
- Magisk 설치 완료
- LSPosed 설치 완료
- Latebris APK 설치 완료

### 5-2. Latebris 모듈 활성화

1. LSPosed Manager 실행
2. **Modules** 탭 이동
3. `Latebris` 모듈 확인
4. 모듈 활성화

### 5-3. 테스트 대상 앱을 scope에 추가

이 단계가 매우 중요합니다.

1. LSPosed에서 Latebris 모듈 상세 화면 열기
2. Scope 또는 적용 대상 앱 목록으로 이동
3. **실제로 테스트할 앱**을 선택
4. 필요하면 앱 강제 종료 후 다시 실행

예를 들어:

- 테스트하려는 대상 앱이 `com.company.app`이면
- LSPosed scope에 `com.company.app`을 추가해야 합니다.

Latebris 앱만 설치되어 있고 테스트 대상 앱이 scope에 없으면, 보통 아래 현상이 나타납니다.

- Latebris 내부 UI는 정상
- Start / Stop도 정상
- 하지만 외부 앱의 Wi-Fi / Cell API 결과는 바뀌지 않음

### 5-4. Latebris 내부 설정 확인

앱 안에서 아래를 확인합니다.

- Master switch = Enabled
- GPS mock = On
- Wi-Fi mock = On
- Cell mock = On
- 선택한 프리셋에 Wi-Fi / Cell 데이터가 실제로 들어 있음

### 5-5. 전체 기능 테스트 순서

1. Latebris 실행
2. 프리셋 선택
3. Start 누르기
4. 상태가 `Running`으로 바뀌는지 확인
5. 테스트 대상 앱 실행
6. 대상 앱이 호출하는 Wi-Fi / Cell 결과가 프리셋 기반으로 바뀌는지 확인

## 7. 자주 헷갈리는 부분

### Q1. 순정폰인데 GPS도 아예 안 되는 것 같다

아래를 순서대로 봅니다.

1. 위치 권한이 허용됐는지
2. 개발자 옵션이 열려 있는지
3. 모의 위치 앱 선택이 필요한 기기인지
4. OEM 정책이 mock location을 더 엄격하게 막고 있지 않은지

### Q2. 루팅은 했는데 Wi-Fi / Cell 값이 안 바뀐다

가장 먼저 아래를 확인합니다.

1. LSPosed가 설치되어 있는지
2. Latebris 모듈이 활성화되어 있는지
3. 테스트 대상 앱이 scope에 추가되어 있는지
4. 프리셋에 Wi-Fi / Cell 값이 실제로 들어 있는지

### Q3. Start는 되는데 외부 앱 결과가 안 바뀐다

이 경우는 대부분 아래 둘 중 하나입니다.

- LSPosed scope 누락
- 대상 앱이 Latebris 후킹 경로 밖에 있음

즉, Latebris 내부 상태가 `Running`이라고 해서 외부 앱 결과가 반드시 바뀌는 것은 아닙니다.

## 8. 추천 시작 순서

처음에는 아래처럼 가는 것이 가장 안전합니다.

1. **비루팅 순정폰 모드로 먼저 앱 UI와 GPS 흐름 확인**
2. 그 다음 테스트 전용 기기에서 **루팅 + Magisk + LSPosed** 준비
3. 마지막으로 **테스트 대상 앱 scope 추가 후 Wi-Fi / Cell 전체 기능 검증**

이 순서로 가면 어디서 문제가 생겼는지 구분하기 훨씬 쉽습니다.
